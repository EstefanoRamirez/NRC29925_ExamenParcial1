Scrollspy y Tabs - Grupo 03

TEORÍA
¿Qué es Scrollspy?

Scrollspy es un plugin de Bootstrap que permite actualizar automáticamente los enlaces de navegación basándose en la posición del scroll en la página. Es especialmente útil para crear tablas de contenido (table of contents) que se actualizan dinámicamente mientras el usuario se desplaza por el documento.

¿Qué son las Tabs?

Las Tabs (Pestañas) son componentes de Bootstrap que permiten organizar contenido en múltiples paneles, mostrando solo uno a la vez. Los usuarios pueden alternar entre diferentes secciones haciendo clic en las pestañas correspondientes, lo que mejora la organización y la experiencia del usuario al evitar el desplazamiento excesivo.

Características Principales:
  • Scrollspy: Actualiza automáticamente los enlaces de navegación según la posición del scroll
  • Data Attributes: Se configura mediante atributos HTML como data-spy="scroll", data-target y data-offset
  • Tabs: Permite organizar contenido en múltiples paneles intercambiables
  • Navegación Dinámica: Los elementos de navegación se resaltan automáticamente cuando su sección correspondiente está visible
  • Responsive: Funciona perfectamente en dispositivos móviles y tablets
Atributos Clave de Scrollspy:
  • data-spy="scroll": Activa el plugin Scrollspy en el elemento
  • data-target="#menu": Especifica el ID del elemento de navegación que se actualizará
  • data-offset="0": Define el offset en píxeles desde la parte superior antes de activar un elemento

Explicación Detallada del Código
1. Estructura HTML de las Tabs

Las tabs se crean usando la clase nav nav-tabs de Bootstrap. Cada pestaña tiene un atributo data-bs-toggle="tab" (en Bootstrap 5) y un enlace que apunta al ID del panel correspondiente. También se incluyen atributos de accesibilidad como role y aria-*. 

<ul class="nav nav-tabs" role="tablist">
  <li class="nav-item" role="presentation">
    <a class="nav-link active" data-bs-toggle="tab" href="#django">Django</a>
  </li>
  <li class="nav-item" role="presentation">
    <a class="nav-link" data-bs-toggle="tab" href="#bootstrap">Bootstrap</a>
  </li>
</ul>
2. Contenido de los Paneles

Cada panel de contenido se define con la clase tab-pane y debe tener un ID que coincida con el href de la pestaña correspondiente. El panel activo por defecto debe tener las clases fade show active. También se incluye el atributo role="tabpanel" para accesibilidad. 

<div class="tab-content">
  <div class="tab-pane fade show active" id="django" role="tabpanel">
    <!-- Contenido del panel -->
  </div>
  <div class="tab-pane fade" id="bootstrap" role="tabpanel">
    <!-- Contenido del panel -->
  </div>
</div>
3. Implementación de Scrollspy

Para activar Scrollspy, se agregan los atributos data-spy="scroll", data-target="#menu" y data-offset="0" al elemento contenedor del contenido desplazable. El menú de navegación debe tener enlaces que apunten a los IDs de las secciones. 

<div class="col-md-9 contenido-scrollspy" 
     data-spy="scroll" 
     data-target="#menu" 
     data-offset="0">
  <div class="panel panel-primary" id="dj1">
    <!-- Contenido -->
  </div>
</div>
<div class="col-md-3" id="menu">
  <ul class="nav nav-pills nav-stacked">
    <li><a href="#dj1">Sección 1</a></li>
  </ul>
</div>
4. Estilos CSS Personalizados

Se define una altura fija y scroll para el contenedor con Scrollspy, permitiendo que el plugin funcione correctamente al detectar las secciones visibles. 

.contenido-scrollspy {
  overflow-y: scroll;
  height: 550px;
}
5. Inicialización con JavaScript

Bootstrap 5 inicializa automáticamente los componentes cuando se cargan los scripts mediante los atributos data-bs-*. Para Scrollspy, se puede usar JavaScript personalizado con Intersection Observer API para detectar las secciones visibles y actualizar el menú de navegación automáticamente. 

<script src="js/vendor/bootstrap.bundle.min.js"></script>
<script>
  // Scrollspy se inicializa automáticamente o manualmente
  var scrollSpy = new bootstrap.ScrollSpy(document.body, {
    target: '#menu',
    offset: 0
  });
</script>

Introducción a Django

Django es un framework web de alto nivel escrito en Python que fomenta el desarrollo rápido y el diseño limpio y pragmático. Fue diseñado para ayudar a los desarrolladores a tomar aplicaciones desde el concepto hasta la finalización lo más rápido posible.

Django se encarga de gran parte de las complicaciones del desarrollo web, por lo que puedes concentrarte en escribir tu aplicación sin necesidad de reinventar la rueda. Es gratuito y de código abierto.

Instalación de Django

Para instalar Django, puedes usar pip, el gestor de paquetes de Python. El comando básico es:

pip install django

Una vez instalado, puedes verificar la instalación ejecutando:

django-admin --version
Óptimo Entorno de Desarrollo

Para un entorno de desarrollo óptimo con Django, se recomienda:

  • Usar entornos virtuales (virtualenv o venv)
  • Configurar un IDE adecuado (PyCharm, VS Code, etc.)
  • Utilizar un sistema de control de versiones (Git)
  • Configurar una base de datos adecuada para desarrollo
Conexión de Django con PostgreSQL

Para conectar Django con PostgreSQL, necesitas instalar el adaptador psycopg2:

pip install psycopg2-binary

Luego, configura la base de datos en settings.py con los parámetros de conexión apropiados.

Diseño de Aplicaciones

Django sigue el patrón de diseño MVT (Model-View-Template). Los modelos definen la estructura de datos, las vistas manejan la lógica de negocio, y los templates se encargan de la presentación.

Introducción a Bootstrap 3

Bootstrap es un framework front-end de código abierto para diseñar sitios web y aplicaciones web. Contiene plantillas de diseño basadas en HTML y CSS para tipografía, formularios, botones, navegación y otros componentes de interfaz.

Desarrollado originalmente por Twitter, Bootstrap se ha convertido en uno de los frameworks más populares del mundo, facilitando el desarrollo de interfaces web responsivas y modernas. Su filosofía se basa en proporcionar herramientas que permitan a los desarrolladores crear interfaces consistentes y profesionales de manera rápida y eficiente.

Bootstrap incluye un sistema de grid responsivo, componentes reutilizables, plugins de JavaScript y utilidades CSS que simplifican significativamente el proceso de desarrollo web. Es compatible con todos los navegadores modernos y se actualiza constantemente para mantenerse al día con las mejores prácticas del desarrollo web.

Filosofía Mobile First

Bootstrap 3 adopta un enfoque "mobile-first", lo que significa que el diseño se optimiza primero para dispositivos móviles y luego se escala para pantallas más grandes. Esta metodología garantiza que las aplicaciones se vean y funcionen perfectamente en todos los dispositivos, desde smartphones hasta monitores de escritorio.

El enfoque mobile-first implica que los estilos base se diseñan para pantallas pequeñas, y luego se utilizan media queries para agregar estilos adicionales a medida que aumenta el tamaño de la pantalla. Esto es más eficiente que el enfoque tradicional de diseñar para desktop y luego reducir para móviles.

Esta filosofía se refleja en el sistema de grid de Bootstrap, donde las clases por defecto aplican estilos para móviles, y las clases con prefijos como sm, md, lg y xl se aplican en pantallas más grandes. Esto asegura una experiencia óptima en todos los dispositivos.

Mockups del Proyecto

Los mockups son esenciales para planificar el diseño antes de comenzar el desarrollo. Ayudan a visualizar la estructura y el flujo de la aplicación, permitiendo identificar problemas de diseño temprano en el proceso.

Con Bootstrap, puedes crear mockups rápidamente usando sus componentes predefinidos, lo que acelera significativamente el proceso de diseño y desarrollo. Los componentes de Bootstrap como cards, modals, navbars y forms proporcionan una base sólida para crear prototipos funcionales.

Bootstrap también ofrece temas y plantillas predefinidas que pueden servir como punto de partida para tus proyectos. Estas plantillas incluyen ejemplos de layouts comunes, lo que te permite comenzar rápidamente y personalizar según tus necesidades específicas.

Primeros Pasos

Para comenzar con Bootstrap, simplemente incluye los archivos CSS y JS en tu proyecto HTML. Puedes descargarlo desde el sitio oficial o usar un CDN para una carga más rápida:

<!-- CSS de Bootstrap -->
<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.1.3/dist/css/bootstrap.min.css" rel="stylesheet">

<!-- JavaScript de Bootstrap (incluye Popper) -->
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.1.3/dist/js/bootstrap.bundle.min.js"></script>

Una vez incluido, puedes comenzar a usar todas las clases y componentes de Bootstrap en tu HTML. Bootstrap utiliza clases CSS para aplicar estilos, lo que hace que sea muy fácil de usar. Por ejemplo, puedes crear un botón simplemente agregando la clase btn btn-primary a un elemento button.

Bootstrap también requiere jQuery para algunos de sus componentes JavaScript, aunque en Bootstrap 5 esto ha cambiado y ahora usa JavaScript vanilla. Asegúrate de verificar la versión de Bootstrap que estás usando para conocer los requisitos específicos.

Grid System

El sistema de grid de Bootstrap utiliza 12 columnas y permite crear diseños responsivos de manera sencilla mediante clases predefinidas. Las clases de grid como col-md-6, col-lg-4 permiten controlar el ancho de los elementos en diferentes tamaños de pantalla, facilitando la creación de layouts adaptativos.

El grid se basa en flexbox, lo que proporciona un control más preciso sobre el diseño. Cada fila puede contener hasta 12 columnas, y puedes combinar diferentes clases para crear layouts complejos. Por ejemplo, col-md-6 significa que el elemento ocupará 6 de las 12 columnas (50% del ancho) en pantallas medianas y superiores.

Bootstrap también incluye clases de utilidad para offsets, ordenamiento y alineación, lo que te da control total sobre la disposición de tus elementos. El sistema de grid es completamente responsivo y se adapta automáticamente a diferentes tamaños de pantalla, haciendo que sea fácil crear diseños que se vean bien en todos los dispositivos.

Introducción a Python

Python es un lenguaje de programación de alto nivel, interpretado y de propósito general. Su filosofía de diseño enfatiza la legibilidad del código con el uso notable de espacios en blanco significativos. Python fue creado por Guido van Rossum y fue lanzado por primera vez en 1991.

Python es ampliamente utilizado en desarrollo web, ciencia de datos, inteligencia artificial, automatización y muchas otras áreas debido a su sintaxis clara y su gran ecosistema de bibliotecas. El lenguaje es conocido por su simplicidad y facilidad de aprendizaje, lo que lo convierte en una excelente opción tanto para principiantes como para desarrolladores experimentados.

Una de las características más destacadas de Python es su filosofía "Zen de Python", que enfatiza principios como la simplicidad, la legibilidad y la elegancia del código. Python soporta múltiples paradigmas de programación, incluyendo programación orientada a objetos, funcional e imperativa, lo que lo hace extremadamente versátil.

Instalación de Python

Para instalar Python, puedes descargarlo desde el sitio oficial python.org o usar un gestor de paquetes según tu sistema operativo. Python está disponible para Windows, macOS y Linux, y el proceso de instalación varía según la plataforma.

# En Windows, descarga el instalador desde python.org
# Ejecuta el instalador y marca la opción "Add Python to PATH"

# En Linux (Ubuntu/Debian):
sudo apt-get update
sudo apt-get install python3 python3-pip

# En macOS:
brew install python3

# Verificar instalación:
python3 --version
pip3 --version

Es importante verificar que Python se haya instalado correctamente ejecutando python --version o python3 --version en la terminal. También asegúrate de que pip, el gestor de paquetes de Python, esté instalado y funcionando correctamente.

Entornos Virtuales

Los entornos virtuales permiten aislar las dependencias de un proyecto, evitando conflictos entre diferentes proyectos que pueden requerir versiones diferentes de las mismas bibliotecas. Python incluye el módulo venv para crear entornos virtuales.

# Crear un entorno virtual:
python3 -m venv nombre_entorno

# Activar en Windows:
nombre_entorno\Scripts\activate

# Activar en Linux/Mac:
source nombre_entorno/bin/activate

# Desactivar (en cualquier sistema):
deactivate

Una vez activado el entorno virtual, cualquier paquete que instales con pip se instalará solo en ese entorno, no globalmente. Esto es esencial para mantener las dependencias organizadas y evitar conflictos entre proyectos. También es una buena práctica incluir el directorio del entorno virtual en tu archivo .gitignore si usas control de versiones.

Gestión de Paquetes con pip

pip es el gestor de paquetes estándar de Python. Permite instalar, actualizar y gestionar bibliotecas de terceros de manera sencilla. pip viene incluido con Python desde la versión 3.4, aunque puede que necesites instalarlo por separado en algunas distribuciones de Linux.

# Instalar un paquete:
pip install nombre_paquete

# Instalar una versión específica:
pip install nombre_paquete==1.2.3

# Actualizar un paquete:
pip install --upgrade nombre_paquete

# Ver paquetes instalados:
pip list

# Guardar dependencias en requirements.txt:
pip freeze > requirements.txt

# Instalar desde requirements.txt:
pip install -r requirements.txt

El archivo requirements.txt es fundamental para cualquier proyecto Python, ya que permite recrear el entorno de desarrollo en cualquier máquina. Siempre es buena práctica mantener este archivo actualizado y compartirlo con tu equipo o incluirlo en tu repositorio de código.

Estructura de un Proyecto Python

Una estructura típica de proyecto Python sigue convenciones que facilitan la organización y el mantenimiento del código. Aunque no hay una estructura única obligatoria, hay patrones comunes que se recomiendan seguir:

mi_proyecto/
├── README.md              # Documentación del proyecto
├── requirements.txt       # Dependencias del proyecto
├── setup.py              # Configuración para distribución (opcional)
├── src/                  # Código fuente principal

│       ├── __init__.py
│       ├── modulo1.py
│       └── modulo2.py
├── tests/                # Tests unitarios
│   └── test_modulo1.py
└── docs/                 # Documentación adicional
    └── guia.md

Esta estructura ayuda a mantener el código organizado, facilita las pruebas y hace que el proyecto sea más fácil de entender para otros desarrolladores. El archivo __init__.py es necesario para que Python reconozca un directorio como un paquete, y el archivo README.md debe contener información sobre cómo instalar, configurar y usar el proyecto.

Introducción a Responsive Web Design

Responsive Web Design (RWD) es un enfoque de diseño web que hace que las páginas web se vean y funcionen bien en una variedad de dispositivos y tamaños de pantalla, desde smartphones hasta monitores de escritorio de alta resolución.

El diseño responsivo utiliza técnicas como media queries, imágenes flexibles y layouts fluidos para adaptar el contenido a diferentes resoluciones de pantalla. Este enfoque se ha convertido en esencial en el desarrollo web moderno, ya que los usuarios acceden a sitios web desde una amplia gama de dispositivos.

El concepto de diseño responsivo fue popularizado por Ethan Marcotte en 2010, y desde entonces se ha convertido en un estándar de la industria. Un diseño responsivo no solo mejora la experiencia del usuario, sino que también puede mejorar el SEO y reducir los costos de mantenimiento al eliminar la necesidad de mantener versiones separadas del sitio para diferentes dispositivos.

Media Queries

Las media queries son la base del diseño responsivo. Permiten aplicar estilos CSS según las características del dispositivo, como el ancho de la pantalla, la orientación o la resolución. Las media queries son parte de la especificación CSS3 y son ampliamente soportadas por los navegadores modernos.

/* Ejemplo básico de media query */
@media (max-width: 768px) {
  .container {
    width: 100%;
    padding: 10px;
  }
  
  .sidebar {
    display: none;
  }
}

/* Media query para tablets */
@media (min-width: 769px) and (max-width: 1024px) {
  .container {
    width: 750px;
    margin: 0 auto;
  }
}

/* Media query para desktop */
@media (min-width: 1025px) {
  .container {
    width: 1200px;
    margin: 0 auto;
  }
}

Las media queries permiten crear diseños que se adaptan dinámicamente a diferentes tamaños de pantalla, lo que es fundamental para el diseño responsivo. Puedes combinar múltiples condiciones y usar diferentes características del dispositivo para crear diseños altamente personalizados.

Viewport Meta Tag

El viewport meta tag es esencial para el diseño responsivo. Se coloca en el <head> del HTML y le dice al navegador cómo controlar las dimensiones y el escalado de la página en dispositivos móviles. Sin este tag, los navegadores móviles pueden renderizar la página como si fuera para desktop, haciendo que se vea muy pequeña.

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Los atributos más comunes del viewport incluyen:

  • width=device-width: Establece el ancho del viewport al ancho del dispositivo
  • initial-scale=1.0: Establece el nivel de zoom inicial
  • maximum-scale=1.0: Previene el zoom del usuario (usar con precaución)
  • user-scalable=no: Desactiva el zoom del usuario (no recomendado para accesibilidad)

Este tag es crucial para que el diseño responsivo funcione correctamente en dispositivos móviles, ya que sin él, los navegadores móviles pueden aplicar un zoom automático que distorsiona el diseño.

Imágenes Responsivas

Para hacer las imágenes responsivas, es importante usar técnicas que permitan que las imágenes se escalen correctamente sin desbordarse del contenedor. Hay varias formas de lograr esto, tanto con CSS como con atributos HTML modernos.

/* Método 1: CSS básico */
img {
  max-width: 100%;
  height: auto;
  display: block;
}

/* Método 2: Usando la clase de Bootstrap */
<img src="imagen.jpg" class="img-fluid" alt="Descripción">

/* Método 3: Usando srcset para diferentes resoluciones */
<img src="imagen-small.jpg" 
     srcset="imagen-small.jpg 480w, 
             imagen-medium.jpg 768w, 
             imagen-large.jpg 1200w"
     sizes="(max-width: 480px) 100vw, 
            (max-width: 768px) 50vw, 
            33vw"
     alt="Descripción">

El uso de max-width: 100% asegura que las imágenes nunca excedan el ancho de su contenedor, mientras que height: auto mantiene la proporción de aspecto. Para un mejor rendimiento, considera usar el atributo srcset para servir diferentes tamaños de imagen según el dispositivo del usuario.

Breakpoints Comunes

Los breakpoints son puntos donde el diseño cambia para adaptarse a diferentes tamaños de pantalla. Aunque no hay breakpoints universales, hay rangos comunes que se usan ampliamente en el diseño responsivo. Estos breakpoints ayudan a crear diseños que se adaptan a los dispositivos más comunes.

/* Breakpoints comunes */
/* Extra small devices (phones, menos de 576px) */
@media (max-width: 575.98px) { }

/* Small devices (tablets verticales, 576px y más) */
@media (min-width: 576px) and (max-width: 767.98px) { }

/* Medium devices (tablets horizontales, 768px y más) */
@media (min-width: 768px) and (max-width: 991.98px) { }

/* Large devices (desktops, 992px y más) */
@media (min-width: 992px) and (max-width: 1199.98px) { }

/* Extra large devices (large desktops, 1200px y más) */
@media (min-width: 1200px) { }

Estos breakpoints corresponden aproximadamente a:

  • Mobile: hasta 576px (smartphones)
  • Tablet (vertical): 576px - 768px
  • Tablet (horizontal): 768px - 992px
  • Desktop: 992px - 1200px
  • Large Desktop: 1200px y superior

Estos breakpoints son similares a los usados por frameworks como Bootstrap, lo que facilita la integración con estas herramientas. Sin embargo, siempre es importante probar tu diseño en dispositivos reales para asegurar que funcione correctamente.

Introducción a Node.js

Node.js es un entorno de ejecución de JavaScript construido sobre el motor V8 de Chrome. Permite ejecutar JavaScript en el servidor, abriendo nuevas posibilidades para el desarrollo web. Node.js fue creado por Ryan Dahl en 2009 y ha revolucionado el desarrollo backend con JavaScript.

Node.js es ideal para aplicaciones en tiempo real, APIs, microservicios y aplicaciones que requieren alta concurrencia gracias a su modelo de I/O no bloqueante basado en eventos. Este modelo permite que Node.js maneje miles de conexiones simultáneas de manera eficiente, lo que lo hace perfecto para aplicaciones como chats, juegos en línea y sistemas de streaming.

Una de las principales ventajas de Node.js es que permite usar el mismo lenguaje (JavaScript) tanto en el frontend como en el backend, lo que simplifica el desarrollo y permite compartir código entre cliente y servidor. Node.js también tiene un ecosistema masivo de paquetes disponibles a través de npm, el gestor de paquetes más grande del mundo.

Instalación de Node.js

Para instalar Node.js, puedes descargar el instalador desde nodejs.org o usar un gestor de versiones como nvm (Node Version Manager), que es especialmente útil si necesitas trabajar con múltiples versiones de Node.js en diferentes proyectos.

nvm use node

# En Windows, descarga nvm-windows desde GitHub

# Verificar instalación:
node --version
npm --version

# Instalación directa desde nodejs.org
# Descarga el instalador para tu sistema operativo
# y sigue las instrucciones del instalador

npm (Node Package Manager) viene incluido con Node.js y es esencial para gestionar dependencias. Una vez instalado Node.js, puedes verificar que npm esté funcionando correctamente ejecutando npm --version. Es recomendable usar nvm para gestionar versiones de Node.js, especialmente si trabajas en múltiples proyectos que pueden requerir diferentes versiones.

Gestión de Paquetes con npm

npm es el gestor de paquetes de Node.js y el registro de paquetes más grande del mundo. Permite instalar, actualizar y gestionar dependencias de manera sencilla. npm viene incluido con Node.js, por lo que no necesitas instalarlo por separado.

# Inicializar un proyecto:
npm init
npm init -y  # Con valores por defecto

# Instalar paquetes:
npm install nombre_paquete           # Instalar localmente
npm install -g nombre_paquete        # Instalar globalmente
npm install nombre_paquete --save    # Guardar en dependencies
npm install nombre_paquete --save-dev # Guardar en devDependencies

# Actualizar paquetes:
npm update nombre_paquete
npm update  # Actualizar todos

# Ejecutar scripts:
npm start
npm test
npm run nombre_script

# Ver información:
npm list          # Ver paquetes instalados

El archivo package.json es el corazón de cualquier proyecto Node.js. Contiene la configuración del proyecto, scripts, dependencias y metadatos. Este archivo permite que otros desarrolladores instalen todas las dependencias necesarias simplemente ejecutando npm install.

Crear un Servidor Básico

Node.js incluye módulos integrados que permiten crear servidores HTTP sin necesidad de frameworks externos. Aunque para aplicaciones complejas se recomienda usar frameworks como Express.js, es útil entender cómo funciona un servidor básico.

// server.js
const http = require('http');
const fs = require('fs');
const path = require('path');

const server = http.createServer((req, res) => {
  // Determinar el tipo de contenido
  let contentType = 'text/html';
  let filePath = './index.html';
  
  if (req.url === '/') {
    filePath = './index.html';
  } else if (req.url === '/about') {
    filePath = './about.html';
  }
  
  // Leer y servir el archivo
  fs.readFile(filePath, (err, content) => {
    if (err) {
      res.writeHead(404);
      res.end('Página no encontrada');
    } else {
      res.writeHead(200, {'Content-Type': contentType});
      res.end(content);
    }
  });
});

const PORT = 3000;
server.listen(PORT, () => {
  console.log(`Servidor corriendo en http://localhost:${PORT}`);
});

Este ejemplo muestra cómo crear un servidor HTTP básico que puede servir archivos HTML estáticos. Para aplicaciones más complejas que requieren routing, middlewares, manejo de formularios y otras características avanzadas, se recomienda usar frameworks como Express.js, que simplifican significativamente el desarrollo.

Express.js Framework

Express.js es el framework web más popular para Node.js. Facilita la creación de APIs y aplicaciones web proporcionando una capa delgada de características fundamentales para aplicaciones web y móviles. Express simplifica el manejo de rutas, middlewares y respuestas HTTP.

// Instalar Express:
// npm install express

const express = require('express');
const app = express();
const PORT = 3000;

// Middleware para parsear JSON
app.use(express.json());
app.use(express.urlencoded({ extended: true }));

// Rutas
app.get('/', (req, res) => {
  res.send('Página de inicio');
});

app.get('/api/usuarios', (req, res) => {
  res.json({ usuarios: ['Juan', 'María', 'Pedro'] });
});

app.post('/api/usuarios', (req, res) => {
  const nuevoUsuario = req.body;
  // Lógica para guardar usuario
  res.status(201).json({ mensaje: 'Usuario creado', usuario: nuevoUsuario });
});

// Iniciar servidor
app.listen(PORT, () => {
  console.log(`Servidor Express corriendo en http://localhost:${PORT}`);
});

Express proporciona una API robusta para crear servidores web y APIs RESTful. Incluye características como routing, middlewares, manejo de plantillas, y soporte para múltiples motores de plantillas. Express es minimalista y flexible, permitiendo agregar solo las funcionalidades que necesitas mediante middlewares y plugins adicionales.

Explicación del Código Principal
1. Estructura HTML Base

<body

  <!-- Contenido de la página -->

2. Menú lateral (Tabla de Contenido)

El atributo data-target="#table-of-contents" del body apunta a este menú lateral. Es una lista vertical con enlaces a cada sección del sitio: header, menú, información, video, footer e integrantes. Bootstrap 3 usa .nav, .nav-pills y .nav-stacked para este tipo de menú. 

<aside class="col-lg-3 col-md-3 col-sm-12 left-menu">

    <ul class="nav nav-pills nav-stacked">
      <li class="active"><a href="#header">Header</a></li>
      <li><a href="#menu">Menú</a></li>
      <li><a href="#info">Información</a></li>
      <li><a href="#video-section">Video</a></li>
      <li><a href="#footer">Footer</a></li>
      <li><a href="#tabla">Integrantes</a></li>
    </ul>
  </nav>
</aside>
3. Secciones que Scrollspy va marcando

Cada enlace del menú anterior apunta a una sección que tiene ese mismo id. Al hacer scroll, Bootstrap agrega la clase active al elemento del menú cuyo href coincide con la sección visible. 

<header class="row" id="header">
  <!-- Header con el logo -->
</header>

5. Inicialización con JavaScript

En el index se usa jQuery para mostrar/ocultar la tabla del Grupo 3. Cuando se muestra contenido nuevo que cambia el alto de la página, se puede pedir a Bootstrap que vuelva a calcular el scrollspy. 

<script>
$(document).ready(function () {
  // ocultar tabla al inicio
  $('#tablaGrupo5').hide();

  // mostrar/ocultar tabla al hacer clic en "Grupo3"
  $('#grupo3Link').click(function (e) {
    e.preventDefault();
    $('#tablaGrupo3').slideToggle(400, function () {
      if ($('#tablaGrupo3').is(':visible')) {
        $('html, body').animate({
          scrollTop: $('#tablaGrupo3').offset().top - 20
        }, 500);
      }
      // refrescar scrollspy por si cambió el alto
      $('body').scrollspy('refresh');
    });
  });
});
</script>
