Validación de Formularios - Grupo 09

1) Teoría
Conceptos Fundamentales

La validación de formularios es un proceso fundamental en el desarrollo web que consiste en verificar que los datos ingresados por el usuario sean correctos, coherentes y cumplan con los requisitos establecidos antes de ser enviados al servidor. Su propósito principal es evitar errores, mejorar la experiencia del usuario y garantizar la integridad de la información.

Existen dos tipos de validación:

  • Validación del lado del cliente: se realiza directamente en el navegador del usuario mediante lenguajes como JavaScript o jQuery. Permite detectar errores de forma rápida y visual sin necesidad de recargar la página.
  • Validación del lado del servidor: ocurre una vez que los datos se envían al servidor, asegurando que la información sea segura y cumpla con los parámetros esperados por la aplicación.
Bootstrap y su integración con jQuery

Bootstrap, además de ser un framework CSS para el diseño visual de páginas web, proporciona clases específicas para la validación de formularios. Cuando se combina con jQuery, se logra una validación no solo funcional sino también visualmente atractiva e intuitiva para el usuario.

Clases de Validación
.is-valid

Resalta los campos validados correctamente con color verde.

.is-invalid

Marca los campos con errores en color rojo.

.valid-feedback y .invalid-feedback

Muestran mensajes descriptivos debajo del campo.

jQuery
$(document).ready(function() {
    $('#ejemploForm').on('submit', function(e) {
        e.preventDefault();
        var input = $('#nombre');
        
        if (input[0].checkValidity()) {
            input.addClass('is-valid');
            alert('¡Formulario válido!');
        } else {
            input.addClass('is-invalid');
        }
    });
});

Consideraciones y buenas prácticas:

  • Combinar validación del lado del cliente (UX) con la validación del lado del servidor (seguridad).
  • Mostrar mensajes claros y accesibles para usuarios y lectores de pantalla.
  • Evitar confiar únicamente en clases visuales; usar atributos aria-* cuando sea necesario.

    $('.needs-validation').each(function () {
        $(this).on('submit', function (event) {
            let form = this;

            if (!form.checkValidity()) {
                event.preventDefault();
                event.stopPropagation();
            } else {
                alert('FORM VALIDADO');
            }

            $(form).addClass('was-validated');
        });
    });
});

Explicación de la Validación de Formularios

La validación del formulario embebido se logra mediante una combinación de HTML y JavaScript. A continuación se detallan sus componentes clave con las partes específicas del código donde se encuentran.

HTML del Formulario
  • Contenedor del Formulario: El formulario tiene un ID único (formularioValidacionModal) y el atributo novalidate.
    • El id es crucial para que el JavaScript lo identifique y manipule.
    • El atributo novalidate es fundamental: inhabilita la validación automática del navegador, permitiendo aplicar nuestra propia validación personalizada usando JavaScript.
    • Código HTML: <form id="formularioValidacionModal" novalidate>
  • Campos del Formulario: Campos como Nombre, Usuario, Contraseñas y Género usan el atributo required, indicando que son obligatorios. El campo select para Género tiene una opción inicial (--Seleccione--) deshabilitada y seleccionada por defecto, lo que lo convierte en un campo obligatorio efectivo.
    • Código HTML Nombre: <input type="text" id="nombreModal" name="nombreModal" class="form-control" placeholder="Ingrese el nombre..." required>
    • Código HTML Usuario: <input type="text" id="usuarioModal" name="usuarioModal" class="form-control" placeholder="Ingrese el usuario..." required>
    • Código HTML Contraseña: <input type="password" id="pass1Modal" name="pass1Modal" class="form-control" placeholder="Ingrese la contraseña..." required>
    • Código HTML Confirmar Contraseña: <input type="password" id="pass2Modal" name="pass2Modal" class="form-control" placeholder="Confirme la contraseña..." required>
    • Código HTML Género: <select class="form-control" id="generoModal" name="generoModal" required>
    • Código HTML Opción Género: <option value="" selected disabled>--Seleccione--</option>
